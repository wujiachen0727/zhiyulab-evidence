# E2：GOGC 影响对比 — 实测数据

> 🟢已验证 — 本机实测，Go 1.26.2 darwin/arm64

## 运行命令

```bash
for gogc in 50 100 200 400 off; do GOGC=$gogc go run e2_gogc_compare.go; done
```

## 实测结果

**场景**：10MB 长期存活对象 + 1000 万次 64B 短生命周期分配

| GOGC | GC 次数 | GC CPU% | 总耗时 | 行为 |
|------|:-------:|:-------:|:------:|------|
| 50 | 4 | 5.3% | 4.8ms | 高频 GC，每次堆到 15MB 就触发 |
| 100 | 2 | 4.8% | 3.4ms | 默认值，平衡 |
| 200 | 1 | 4.4% | 2.6ms | 低频 GC，堆更大 |
| 400 | 0 | 0.0% | 2.6ms | 堆没到触发阈值 |
| off | 0 | 0.0% | 2.4ms | 不 GC，最快但内存不受控 |

## 关键洞察

1. **GOGC=50 比 GOGC=200 多跑了 3 次 GC**，GC CPU 从 4.4% 升到 5.3%，耗时多了 85%
2. **GOGC=400/off 没触发 GC**——因为 10MB 存活堆 × 5 = 50MB，1000 万次 64B 分配总量约 640MB 但对象短命，堆没涨到阈值
3. **GOGC 是"内存换 CPU"的旋钮**：GOGC 小→GC 频繁→CPU 贵但堆小；GOGC 大→GC 懒惰→CPU 省但堆大
