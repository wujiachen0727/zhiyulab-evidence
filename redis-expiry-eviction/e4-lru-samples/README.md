# E4 实验：maxmemory-samples 对 LRU 近似效果的影响

## 目的
验证 maxmemory-samples 参数对 LRU 近似淘汰精度的影响

## 环境
- Redis 7.0 (Docker `redis:7-alpine`)
- redis-cli 8.8.0
- macOS ARM64 (darwin/arm64)

## 实验方法
1. 创建 1000 个 key，访问前 200 个各 10 次（制造热/冷分布）
2. 设置 maxmemory 为 60% 容量，触发淘汰
3. 分别测试 samples=5（默认）、20（高采样）、1（极低采样）

## 关键发现

| samples | 热 key (1-200) 保留 | 冷 key (801-1000) 保留 | DBSIZE | 结论 |
|:-------:|:------------------:|:--------------------:|:------:|------|
| **5**（默认） | 200/200 | 172/200 | 487 | 热 key 全部保留，冷 key 部分淘汰 |
| **20**（高采样） | 200/200 | 194/200 | 476 | 淘汰更精确——更多冷 key 被保留（近似 LRU 更接近精确 LRU） |
| **1**（极低采样） | 96/200 | 92/200 | 481 | 几乎随机——热 key 和冷 key 的保留率接近 |

## 结论
- samples 越大，近似 LRU 越接近精确 LRU，能更好地区分热 key 和冷 key
- samples=1 时近似 LRU 退化为随机淘汰
- Redis 默认 samples=5 是性能与精度的合理平衡
