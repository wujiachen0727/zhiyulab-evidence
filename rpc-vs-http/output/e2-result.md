# 实验 E2：端到端调用延迟分解

> 业务逻辑模拟耗时: 10ms（代表一次中等量级的 DB/下游调用）
> 测量样本: 1000 次调用，HTTP/1.1 + JSON over Keep-Alive

## 各环节耗时（中位数 P50）

| 环节 | 中位耗时 | 占总耗时 |
|------|--------:|--------:|
| 请求序列化 (JSON Marshal)   | 2.792µs | 0.02% |
| 网络栈 + 协议处理            | 1.907959ms | 15.96% |
| 业务逻辑 (DB/下游调用模拟)   | 10ms | 83.64% |
| 响应反序列化 (JSON Unmarshal) | 24.25µs | 0.20% |
| **端到端 P50 总耗时** | **11.955292ms** | 100%!|(MISSING)

- 端到端 P99: 14.458625ms（网络 P99: 4.259292ms）

## 关键推论
- 「协议+序列化」总占比: **16.19%**
- 「业务逻辑」占比:        83.64%

## 假设业务逻辑更重的场景（敏感性分析）
- 业务 20ms → 协议+序列化占比 8.82%
- 业务 50ms → 协议+序列化占比 3.73%
- 业务 100ms → 协议+序列化占比 1.90%
