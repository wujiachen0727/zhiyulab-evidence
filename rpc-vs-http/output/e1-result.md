# 实验 E1：协议层性能对比（同进程，无网络抖动）
# 配置：workers=32, duration=5s
# Go： (see go.mod)

## 端到端 QPS / 延迟

| 方案 | QPS | P50 延迟 | P99 延迟 | 平均响应大小 |
|------|----:|--------:|--------:|------------:|
| HTTP/1.1 + JSON | 104841 | 276µs | 919µs | 179 B |
| HTTP/2   + JSON | 77007 | 404µs | 848µs | 179 B |
| HTTP/2   + 二进制 | 77804 | 403µs | 837µs | 84 B |

## 单纯序列化/反序列化 (50000 次平均)

| 操作 | JSON | 二进制 |
|------|-----:|------:|
| 编码 | 253ns | 115ns |
| 解码 | 1.367µs | 217ns |
| Body 大小 | 179 B | 84 B |

## 推论提示
- HTTP/2 + 二进制 vs HTTP/1.1 + JSON 的 P50 延迟差: -127µs
- HTTP/2 + 二进制 vs HTTP/2 + JSON  的 P50 延迟差: 1µs
- 体积压缩比 (二进制/JSON): 46.9%
