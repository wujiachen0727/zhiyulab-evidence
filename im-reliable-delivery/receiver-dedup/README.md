# E3：接收端多端幂等去重

演示 at-least-once 投递的第三道防线——接收端按 `msgID` 多端幂等去重。

## 运行
```bash
go run .
```
环境：Go 1.26.4 darwin/arm64，仅用标准库。

## 关键结论（见 output/receiver-dedup/result.txt）
- 场景1：同一消息推给一个用户的三台设备（phone/pc/tablet），每台各消费 1 次。
- 场景2：at-least-once 下服务端重推同一条历史消息，去重集识别已消费 → 丢弃，每台最终消费均为 1 条，无重复。
- 核心：每设备各自维护 `seen` 集合；消息要出现在我的所有设备上，但不能在某台设备上出现两次。

## 与"通用 MQ 单消费者幂等"的区分（本文原创锚点）
- **多端维度**：同一消息推给一个用户的 N 台设备，每台设备是独立消费者，各自有消费状态；不是 MQ 单消费者 Redis 去重。
- **历史消息维度**：设备离线后上线可能收到服务端重推的历史消息，去重集保证不重复消费。
- 生产环境 `seen` 集合用 Redis SETNX 或 DB 唯一约束 + 已读游标实现；本 demo 用进程内 map 模拟，保证可独立运行、可复现。

## 与 IM 失败模式的绑定
对应"多端重复"场景：at-least-once 保证消息至少送达一次，接收端去重保证用户看到一次。
