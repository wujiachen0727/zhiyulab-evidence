# E2：服务端 store-and-forward（离线暂存 + 上线拉历史）

演示 at-least-once 投递的第二道防线——服务端落库暂存，离线设备上线后拉取历史消息。

## 运行
```bash
go run .
```
环境：Go 1.26.4 darwin/arm64，仅用标准库。

## 关键结论（见 output/server-store-forward/result.txt）
- 消息先落库（seq=1），在线设备 phone 立即收到推送。
- 离线设备 pc / tablet 暂存，上线时各自从库拉取历史消息，最终三台设备收件箱均为 1 条。
- 核心：offline 期间消息不丢，靠服务端 store-and-forward 兜底。

## 简化说明
真实系统按"每设备已读游标"增量拉取，本 demo 为演示清晰让离线设备拉取全量历史。生产环境不会把已读消息再推一遍。

## 与 IM 失败模式的绑定
对应"设备离线"场景：用户锁屏 / 切后台 / 无网时，消息在服务端落库，待设备上线再投递，消息不丢。
