# E2 实验记录

## 实验名
e2-java-vs-go — 同一功能（日志系统）的 Java 式（继承）vs Go 式（接口组合）实现对比

## 核心对比

| 维度 | Java 式（继承） | Go 式（接口组合） |
|------|:--------------:|:----------------:|
| 输出目标数 | 3 个类 | 3 个 Output 实现 |
| 格式类型数 | 需嵌入每个类 | 2 个 Formatter 实现 |
| 总组合数 | 6 个类（继承链） | 5 个接口/实现（可自由组合） |
| 新增一种输出 | 新建 1 个类 + 复制格式逻辑 | 新建 1 个 Output 实现 |
| 新增一种格式 | 修改所有输出类 | 新建 1 个 Formatter 实现 |
| 测试难度 | 需测试继承链 | 可独立测试每个实现 |

## 结论
Go 的接口组合方式在可组合性、可扩展性、可测试性上全面优于继承方式。关键是"极小接口"的设计——Output 只做输出，Formatter 只做格式化，Logger 组合两者。
