# E4 实验记录

## 实验名
e4-stdlib-patterns — Go 标准库中的设计模式实例解读

## 覆盖模式
| 模式 | 标准库实现 | Go 式特点 |
|------|-----------|----------|
| 单例模式 | sync.Once | 不是"唯一实例"，是"只执行一次" |
| 工厂模式 | NewXxx 函数 | 封装初始化逻辑，不负责"条件创建" |
| 策略模式 | http.Handler | 极小接口（1 个方法）|
| 适配器模式 | http.HandlerFunc | 函数类型适配为接口 |
| 装饰器模式 | io.MultiReader | 包装 Reader 增加功能 |
| 观察者模式 | context.Context | 取消信号传播（非阻塞）|
| 建造者模式 | bytes.Buffer | 流式写入 |

## 关键洞察
Go 标准库不是"实现了设计模式"，而是"设计意图正好匹配了模式"。每个模式在 Go 中的实现都比传统 OOP 语言更简洁——因为 Go 的接口设计（极小接口 + 隐式实现）天然降低了模式的实现成本。
