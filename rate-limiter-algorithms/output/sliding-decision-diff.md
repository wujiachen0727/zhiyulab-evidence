# E2.1 + E2.2 实验输出：滑动窗口内存与决策差异

> [推演] E2.1 基于 Redis ZSET 内存模型公式计算（非 Redis 实测）
> [实测 Go 1.26.2] E2.2 三变种对比

## E2.1 内存测算结果

| QPS | 用户数 | 总内存（SW Log） |
|----:|------:|:--------------:|
| 100 | 1,000 | 366 MB |
| 1,000 | 1,000 | **3.6 GB** |
| 1,000 | 10,000 | **35.8 GB** |
| 10,000 | 1,000 | **35.8 GB** |
| 10,000 | 10,000 | **357.6 GB** |

对比 SW Counter（10 个子桶）：
- 10K QPS × 1K 用户：~80 KB
- **内存差距约 45 万倍**

## E2.2 跨窗口决策差异

场景：窗口=1s, 限额=5, 尾部发3个 + 头部发3个

| 请求# | 偏移 | SW Log | SW Counter | Fixed Window |
|:-----:|:----:|:------:|:----------:|:------------:|
| 1 | 800ms | ✓ | ✓ | ✓ |
| 2 | 850ms | ✓ | ✓ | ✓ |
| 3 | 900ms | ✓ | ✓ | ✓ |
| 4 | 1050ms | ✓ | ✓ | ✓ |
| 5 | 1100ms | ✓ | ✓ | ✓ |
| **6** | **1150ms** | **✗ 拒绝** | **✗ 拒绝** | **✓ 通过** |

**关键差异**：Fixed Window 在窗口边界发生"双倍放行"——前窗口尾部的 3 个 + 新窗口头部的 3 个 = 实际 1 秒内放行 6 个（超额 20%）。
