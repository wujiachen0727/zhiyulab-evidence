=== 关键观察 ===
1. JWT 规范本身允许 alg:none——这是协议层面的决定
2. 多数 JWT 库默认不接受 none，但**手写验证代码**非常容易踩坑
3. 核心防线：永远用白名单（allowedAlgs），不信任 header 里的 alg 值
4. 历史案例：2015-2018 年间多个主流 JWT 库爆出 alg-confusion 漏洞
