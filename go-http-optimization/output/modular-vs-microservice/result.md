# 模块化单体 vs 微服务延迟对比实验

> 环境: [实测 Go go1.26.2 darwin/arm64]
> 说明: 使用 HTTP 代替 gRPC，实际 gRPC 延迟更低，本文对比结果是保守估计
> [推演：模拟业务处理] 每个模块 1ms time.Sleep

## 实验设计

- **业务流程**: 创建订单 → 扣减库存 → 发起支付
- **模块化单体**: 3个模块在同一进程内，直接函数调用（无网络、无序列化）
- **微服务**: 3个独立 HTTP 服务，网关 → 库存服务 → 支付服务（2次 HTTP 跨服务调用）
- **压测参数**: 50 并发, 10000 请求

## 延迟对比

| 指标 | 模块化单体（进程内） | 微服务（HTTP调用） | 差异倍数 |
|------|--------------------|--------------------|---------|
| P50 | 3.23ms | 3.53ms | 1.1x |
| P90 | 3.45ms | 4.08ms | 1.2x |
| P99 | 4.23ms | 8.35ms | 2.0x |
| Min | 3.06ms | 3.22ms | - |
| Max | 12.72ms | 25.98ms | - |

## 吞吐量对比

| 指标 | 模块化单体 | 微服务 |
|------|----------|-------|
| QPS | 15022 | 13249 |
| 总耗时 | 666ms | 755ms |
| QPS 倍数 | 1.1x | - |

## 资源对比

| 指标 | 模块化单体 | 微服务 |
|------|----------|-------|
| 内存增量 | 2.5 MB | 8.4 MB |
| GC 次数 | 79 | 63 |
| 错误数 | 0 | 0 |

## 结论

- P50 延迟：微服务比模块化单体慢 **1.1 倍**
- P99 延迟：微服务比模块化单体慢 **2.0 倍**
- QPS：模块化单体是微服务的 **1.1 倍**
- 额外延迟主要来源：HTTP 序列化/反序列化、网络协议栈开销、连接管理
- 若使用 gRPC 替代 HTTP，差距会缩小但仍显著（gRPC 延迟约为 HTTP 的 1/2~1/3）
