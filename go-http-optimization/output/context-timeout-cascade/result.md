# 3 层 HTTP 调用链超时策略对比实验

> [实测 Go 1.26.2 darwin/arm64] · [推演：模拟处理延迟]  
> 实验代码：`evidence/code/context-timeout-cascade/main.go`

## 实验设计

### 调用链

```
Gateway (1ms) → Service (2ms) → DataLayer (5ms/30ms/80ms)
```

### DataLayer 延迟分布

| 类型 | 占比 | 延迟 |
|------|------|------|
| 正常请求 | 75% | 5ms |
| 慢请求 | 20% | 30ms |
| 极慢请求 | 5% | 80ms |

最小总延迟：1 + 2 + 5 = **8ms**  
最大总延迟：1 + 2 + 80 = **83ms**

### 3 种超时策略

| 策略 | Gateway | Service | 说明 |
|------|---------|---------|------|
| 固定超时 | 15ms | 15ms | 每层相同超时，不考虑上游剩余时间 |
| 递减超时 | 15ms | 12ms | 越往下游越短，上游留余量给自身处理 |
| 自适应超时 | 15ms | 动态（上游剩余 − 3ms） | 基于 `ctx.Deadline()` 动态计算，安全余量 3ms |

### 压测参数

- 并发数：100
- 总请求数：20,000
- 客户端超时：200ms

## 实验结果

### 核心指标

| 策略 | P50 | P90 | P99 | 成功率 | 超时错误率 | 级联失败率 | 上游失败率 |
|------|-----|-----|-----|--------|-----------|-----------|-----------|
| 固定超时 | 9.38ms | 16.49ms | 22.80ms | 71.91% | 28.09% | **28.09%** | 0.00% |
| 递减超时 | 8.63ms | 15.40ms | 19.53ms | 73.63% | 26.36% | **1.69%** | 24.68% |
| 自适应超时 | 9.25ms | 16.40ms | 20.77ms | 73.58% | 26.42% | **26.42%** | 0.00% |

### 关键发现

#### 1. 递减超时的级联失败率最低：1.69% vs 28.09%（固定）/ 26.42%（自适应）

递减超时策略让 Service 层的 12ms 超时在 DataLayer 慢请求完成前就触发，Service 立即返回 504。此时 Gateway 还有 3ms 余量（15ms − 12ms），收到 Service 的 504 后直接返回错误，不会等自己超时。这就是**快速失败**——上游尽早返回错误，下游不用浪费等待时间。

#### 2. 固定超时的级联失败率最高：28.09%

每层都是 15ms 超时，Service 和 Gateway 几乎同时超时。Service 超时后 Gateway 也没时间了，导致"双超时"——两层同时触发超时，整条链路彻底崩溃。

#### 3. 自适应超时的级联失败率仍然偏高：26.42%

自适应策略中 Service 的超时 = Gateway 剩余时间 − 3ms。当 Gateway 设 15ms 时，Service 收到请求时剩余约 14ms（扣除 1ms 处理），Service 超时 = 14 − 3 = 11ms。这虽然比固定超时略短，但安全余量 3ms 不够——Service 超时后 Gateway 只剩 3ms，网络传输和序列化往往超过 3ms，Gateway 仍然会级联超时。

#### 4. 递减超时的"上游失败"是特征，不是缺陷

递减超时有 24.68% 的上游失败率（Service/DataLayer 超时传播回 Gateway），但这些请求的 Gateway 层没有超时——Gateway 快速返回了上游错误。与级联失败的区别：

- **级联失败**：Gateway 自身超时 → 客户端等待 15ms 才收到错误 → 体验差
- **上游失败**：Gateway 快速返回上游错误 → 客户端等待 8-12ms 收到明确错误 → 体验好

## 可靠性计算

### 串联系统的可用性衰减

当 N 个服务串联调用时，整体可用性 = 各层可用性的乘积：

```
A_total = A₁ × A₂ × ... × Aₙ = (1 - p)^N
```

其中 `p` 为单层故障率。

### 3 层串联的计算

| 单层可用性 | 单层故障率 | 3 层串联可用性 | 可用性损失 |
|-----------|-----------|--------------|-----------|
| 99.9% | 0.1% | 99.9%³ = **99.700%** | 0.3% |
| 99.5% | 0.5% | 99.5%³ = **98.507%** | 1.5% |
| 99.0% | 1.0% | 99.0%³ = **97.030%** | 3.0% |
| 95.0% | 5.0% | 95.0%³ = **85.738%** | 14.3% |

### N 层串联的可用性衰减趋势

```
N=1:   99.9%^1  = 99.90%
N=3:   99.9%^3  = 99.70%
N=5:   99.9%^5  = 99.50%
N=10:  99.9%^10 = 99.00%
N=20:  99.9%^20 = 98.02%
N=50:  99.9%^50 = 95.12%
```

### 推论

1. **层数越多，可用性衰减越快**：即使单层 99.9%，50 层串联后可用性只剩 95.12%
2. **级联超时放大故障**：固定超时策略下，一层超时会传播为整链超时，实际故障率远高于单层故障率
3. **递减超时是有效的止损手段**：让上游比下游更快超时，实现"快速失败"，避免级联超时将单层故障放大为整链故障
4. **公式验证**：本实验中，DataLayer 的慢请求（≥30ms）占比 25%，在 15ms 超时下必然失败。理论成功率上限 ≈ 75%，实测 71.91%-73.63%，差距来自网络开销和并发抖动
