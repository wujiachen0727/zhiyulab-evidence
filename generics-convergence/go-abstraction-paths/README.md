# Go 抽象路径实验

## 目的

验证同一类集合工具在四条路径下的差异：

1. 复制粘贴：为 `int`、`string` 等类型各写一套。
2. `any/interface{}`：复用函数体，但把类型检查推迟到调用处。
3. 反射：复用函数体，但损失静态可读性和类型直接性。
4. 泛型：复用函数体，并保留编译期类型检查。

## 运行方式

```bash
go run .
go run ./compilefail
```

第二条命令预期失败，用于捕获泛型路径的编译期错误。

## 证据标注

- `go run .` 输出为 `[实测 Go 1.26.2 darwin/arm64]`。
- `go run ./compilefail` 的失败输出为 `[实测 Go 1.26.2 darwin/arm64]`，预期失败，证明错误暴露在编译期。
