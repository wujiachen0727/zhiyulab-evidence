# E4 实验输出 — channel 传切片的所有权转移

**运行命令**：`go run -race main.go`
**Go 版本**：1.26.4 darwin/arm64
**运行时间**：2026-06-19

## 结果

| 模式 | Data Race | 说明 |
|------|:---------:|------|
| 错误：发送后继续使用 | **1 个** | 发送方在 channel 发送后继续写切片，接收方也在写 |
| 正确：所有权转移 | **0 个** | 发送方发送后不再碰切片，所有权完全移交 |

## 关键结论

- channel 传切片时，发送的是切片头（ptr+len+cap）的副本，底层数组还是同一个
- 发送方和接收方同时操作底层数组 → data race
- 解决方式：**发送后不再碰**——把切片的所有权交给接收方
- 这跟 Rust 的所有权规则完全一样——但 Go 没有编译器帮你检查
