# E3 实验输出 — 所有权模型对比

**运行命令**：`go run -race main.go`
**Go 版本**：1.26.4 darwin/arm64
**运行时间**：2026-06-19

## 结果

| 场景 | Data Race | 结果完整性 | 说明 |
|------|:---------:|:----------:|------|
| A：无所有权 | **3 个** | 丢失 3 个元素（7/10） | 所有 goroutine 共享 results |
| B：有所有权 | **0 个** | 完整（10/10） | 只有主 goroutine 写入 results |

## 关键结论

- 同样实现"10 个 worker 处理任务、收集结果"的需求
- 无所有权模型：简洁但不安全，data race 导致数据丢失
- 有所有权模型：用 channel 隔离写入权，零 data race
- 两种方案都没有显式的锁——区别在于"谁拥有数据"的设计决策
